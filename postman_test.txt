### Postman 测试集合 —— 需求流程
###
### 环境变量设置（Postman Environment）:
###   base_url  = http://localhost:8080
###
### 两套独立流程，通过 URL 路径区分：
###   /requirement/requirement_pool/...  → 简版流程（userTask）
###   /requirement/FL25103101/...        → 公司流程（receiveTask）


# ============================================================
# 通用接口（不区分流程）
# ============================================================

### 查询需求列表
POST {{base_url}}/requirement/query
Content-Type: application/json

{"pageNum":1,"pageSize":10}


### 查看需求详情
GET {{base_url}}/requirement/1


### 更新需求
PUT {{base_url}}/requirement/update
Content-Type: application/json

{"id":1,"requirementName":"改名后的需求","priority":"低"}


### 删除需求
DELETE {{base_url}}/requirement/1


### 查看需求流程日志
GET {{base_url}}/requirement/logs?requirementNo=REQ001


# ============================================================
# 流程一：requirement_pool（简版流程）
#
# URL 前缀: /requirement/requirement_pool
# 节点类型: userTask
# 流程走向:
#   开始 → draft(草稿) ─submit→ approval(审批) ─approve→ review(评审)
#                           reject↘          ↙reject
#                                  draft(草稿)
#   评审 ─review_pass→ acceptance(验收) ─accept→ 结束
#        ↙review_reject
#   草稿
#
# 可用事件: submit / approve / reject / review_pass / review_reject / accept
# ============================================================

### 🟢 场景: 完整走通简版流程 ###

### 步骤1: 创建需求 → 自动停在 draft 节点
POST {{base_url}}/requirement/requirement_pool/add
Content-Type: application/json

{
    "requirementNo": "REQ001",
    "requirementType": "IDC",
    "requirementName": "简版流程-完整测试",
    "priority": "高",
    "description": "从草稿一路走到完成"
}
# 响应: currentNode = "draft", status = "草稿"


### 步骤2: 草稿 → 提交审批
POST {{base_url}}/requirement/requirement_pool/flow/send
Content-Type: application/json

{"requirementNo":"REQ001","event":"submit"}
# 响应: currentNode = "approval", status = "待审批"


### 步骤3: 审批 → 通过（进评审）
POST {{base_url}}/requirement/requirement_pool/flow/send
Content-Type: application/json

{"requirementNo":"REQ001","event":"approve"}
# 响应: currentNode = "review", status = "评审中"


### 步骤4: 评审 → 通过（进验收）
POST {{base_url}}/requirement/requirement_pool/flow/send
Content-Type: application/json

{"requirementNo":"REQ001","event":"review_pass"}
# 响应: currentNode = "acceptance", status = "验收中"


### 步骤5: 验收 → 通过（流程结束）
POST {{base_url}}/requirement/requirement_pool/flow/send
Content-Type: application/json

{"requirementNo":"REQ001","event":"accept"}
# 响应: currentNode = "end", status = "已完成"


### 查询状态（随时可查）
GET {{base_url}}/requirement/requirement_pool/flow/state?requirementNo=REQ001


### 🔴 附加: 驳回操作 ###

### 审批阶段驳回（需要先创建另一个需求并 submit 到审批节点）
POST {{base_url}}/requirement/requirement_pool/flow/send
Content-Type: application/json

{"requirementNo":"REQ001","event":"reject"}
# 从审批节点 → 回到 draft


### 评审阶段驳回
POST {{base_url}}/requirement/requirement_pool/flow/send
Content-Type: application/json

{"requirementNo":"REQ001","event":"review_reject"}
# 从评审节点 → 回到 draft


# ============================================================
# 流程二：FL25103101（公司流程）
#
# URL 前缀: /requirement/FL25103101
# 节点类型: receiveTask
# 流程走向:
#   开始 → node_0(新增) ─STAGE→ node_1(草稿) ─EDIT:SUBMIT→ node_45(需求评估)
#                       ─SUBMIT─────────────────────────→ node_45(需求评估)
#
#   需求评估 ─SUBMIT→ node_34(CR拆解)
#           ─REJECT→ node_1(草稿)
#           ─TRANSMIT→ node_45(需求评估)  [转交，留在当前]
#           ─NOTINVOLVED→ node_9(已关闭) → 结束
#
#   CR拆解 ─SUBMIT→ node_51(需求验收)
#          ─REJECT→ node_45(需求评估)     [回退]
#          ─NO-JIRA-SUBMIT→ node_13(已完成) → 结束
#
#   需求验收 ─ACCEPT→ node_13(已完成) → 结束
#
# 可用事件:
#   CRequirementAdd:STAGE / CRequirementAdd:SUBMIT
#   CRequirementEdit:SUBMIT
#   CIRCAssessment:SUBMIT / REJECT / TRANSMIT / NOTINVOLVED
#   CIRCcr:SUBMIT / REJECT / NO-JIRA-SUBMIT
#   CIRCAcceptance:ACCEPT
# ============================================================

### 🟢 场景A: 暂存→编辑→完整走通 ###

### 步骤A1: 创建需求 → 暂存到草稿
POST {{base_url}}/requirement/FL25103101/add
Content-Type: application/json

{
    "requirementNo": "REQ002",
    "requirementType": "IDC",
    "requirementName": "公司流程-先暂存再走完整流程",
    "priority": "高",
    "description": "走完整流程",
    "action": "CRequirementAdd:STAGE"
}
# action 默认就是 STAGE，可以省略
# 响应: currentNode = "node_1"（草稿）


### 步骤A2: 草稿 → 编辑提交（进需求评估）
POST {{base_url}}/requirement/FL25103101/flow/send
Content-Type: application/json

{"requirementNo":"REQ002","event":"CRequirementEdit:SUBMIT"}
# 响应: currentNode = "node_45"（需求评估）


### 步骤A3: 需求评估 → 通过（进CR拆解）
POST {{base_url}}/requirement/FL25103101/flow/send
Content-Type: application/json

{"requirementNo":"REQ002","event":"CIRCAssessment:SUBMIT"}
# 响应: currentNode = "node_34"（CR拆解）


### 步骤A4: CR拆解 → 通过（进需求验收）
POST {{base_url}}/requirement/FL25103101/flow/send
Content-Type: application/json

{"requirementNo":"REQ002","event":"CIRCcr:SUBMIT"}
# 响应: currentNode = "node_51"（需求验收）


### 步骤A5: 需求验收 → 通过（进已完成，结束）
POST {{base_url}}/requirement/FL25103101/flow/send
Content-Type: application/json

{"requirementNo":"REQ002","event":"CIRCAcceptance:ACCEPT"}
# 响应: currentNode = "end", status = "已完成"


### 🟢 场景B: 创建时直接提交（跳过草稿） ###

### 步骤B1: 直接提交到需求评估
POST {{base_url}}/requirement/FL25103101/add
Content-Type: application/json

{
    "requirementNo": "REQ003",
    "requirementType": "IDC",
    "requirementName": "公司流程-直接提交",
    "priority": "高",
    "description": "不经过草稿",
    "action": "CRequirementAdd:SUBMIT"
}
# 响应: currentNode = "node_45"（需求评估）


### 🔴 附加: 驳回 / 转交 / 不涉及 / 回退 ###

### 需求评估 → 驳回（回草稿）
POST {{base_url}}/requirement/FL25103101/flow/send
Content-Type: application/json

{"requirementNo":"REQ003","event":"CIRCAssessment:REJECT"}
# node_45 → node_1（草稿）


### 需求评估 → 转交（留在需求评估）
POST {{base_url}}/requirement/FL25103101/flow/send
Content-Type: application/json

{"requirementNo":"REQ003","event":"CIRCAssessment:TRANSMIT"}
# node_45 → node_45（留在原地）


### 需求评估 → 不涉及（进已关闭）
POST {{base_url}}/requirement/FL25103101/flow/send
Content-Type: application/json

{"requirementNo":"REQ003","event":"CIRCAssessment:NOTINVOLVED"}
# node_45 → node_9（已关闭） → 结束


### CR拆解 → 驳回（回需求评估）
POST {{base_url}}/requirement/FL25103101/flow/send
Content-Type: application/json

{"requirementNo":"REQ002","event":"CIRCcr:REJECT"}
# node_34 → node_45（需求评估）


### CR拆解 → 无需JIRA（直接已完成）
POST {{base_url}}/requirement/FL25103101/flow/send
Content-Type: application/json

{"requirementNo":"REQ002","event":"CIRCcr:NO-JIRA-SUBMIT"}
# node_34 → node_13（已完成） → 结束


### 查询状态（随时可查）
GET {{base_url}}/requirement/FL25103101/flow/state?requirementNo=REQ002
